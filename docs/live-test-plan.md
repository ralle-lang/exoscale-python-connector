# Live test plan ‚Äî by asset type

This document enumerates **every API call the connector exposes**, the live test
sequence that would exercise it, the cost/risk, and the cleanup. It exists so
you can read it once and decide, **per asset type**, what's OK to run against
the test tenant ‚Äî *before* anything is provisioned.

The mocked unit suite (`pytest -m "not integration"`) is already green (163
tests). The mocked layer proves the connector *calls the path it implements*;
this plan covers the live layer that proves the paths match the real API.

---

## Top-level approach (per asset)

For every asset type, the live procedure is:

1. **Mock** ‚Äî already covered by the unit suite. No action.
2. **Provision** ‚Äî create one minimum-cost instance of the resource with a
   safely discriminated name (see [Safety rails](#mandatory-safety-rails)).
3. **Verify** ‚Äî round-trip the create response, `get`, `list`, `find_by_name`,
   model parsing.
4. **Mutate** ‚Äî exercise `update`, `scale`/`resize`, lifecycle actions, and any
   sub-resource ops the connector exposes.
5. **Document** ‚Äî assert each step, capture per-op pass/fail and final report.
6. **Cleanup** ‚Äî delete *only* the resources this run created, matched by the
   safety prefix. Never touches other resources in the tenant.

---

## Mandatory safety rails (apply to every tier)

These are non-negotiable invariants the test harness enforces. They protect the
shared tenant from any cross-contamination.

| # | Rail | Enforcement |
|---|------|-------------|
| 1 | **Naming prefix** `conn-test-<RUNID>-<type>` on every created resource, where `RUNID = <UTC timestamp>-<6 char random>` (e.g. `conn-test-20260601T103015Z-a3f9c2-sg`). | Generated by a single fixture; tests cannot opt out. |
| 2 | **Cleanup whitelist** ‚Äî the teardown helper refuses to delete any resource whose name (or id, when nameless) does not start with `conn-test-`. | Hard `assert` in the teardown helper; the test fails before deletion can run. |
| 3 | **Tracked ids** ‚Äî each test registers every created resource id with a per-session registry; cleanup deletes from the registry, not from a live `list()`. | A wild `list()` could match an unrelated resource. The registry path can't. |
| 4 | **Master mutation switch** ‚Äî `EXOSCALE_ALLOW_MUTATION=1` must be set; absent, mutating tests skip. | Already implemented in `tests/integration/conftest.py`. |
| 5 | **Per-tier opt-in switches** (see [tiers](#tier-model)) ‚Äî each cost or time-sensitive tier needs its own flag. | New fixtures in conftest. |
| 6 | **Run-end sweep** ‚Äî at session end, the harness deletes every id still in the registry (best-effort), then logs anything that survived (orphan candidates). | Pytest session-scope finalizer. |
| 7 | **Orphan garbage collector** (standalone) ‚Äî a separate `scripts/gc_test_resources.py` that lists every asset type, filters by `conn-test-*` *and* age > 1h, and prompts before deletion. Use only if a session crashed mid-way. | Tool, not auto-run. |
| 8 | **No prod fingerprint** ‚Äî there is no hardcoded tenant id in the test code. You set the zone(s) and creds via env vars. (Open-source policy.) | The connector itself enforces this. |

## Tier model

Tests are grouped by cost and risk. Each tier has its own env flag so you can
enable exactly what you want; no flag = skipped.

| Tier | Flag | Cost order | Examples |
|------|------|-----------|----------|
| 0 ‚Äî read-only smoke | `EXOSCALE_RUN_LIVE_TESTS=1` | ‚Ç¨0 | All 18 `list()` calls (already implemented) |
| 1 ‚Äî free mutations | `EXOSCALE_TEST_TIER_1=1` | ‚Ç¨0 | security-group, private-network, anti-affinity-group, ssh-key, iam-role, dns |
| 2 ‚Äî cheap, no compute | `EXOSCALE_TEST_TIER_2=1` | < ‚Ç¨0.01/run | bucket (S3), block-volume + snapshot (detached), elastic-ip |
| 3 ‚Äî compute, minutes-of-runtime | `EXOSCALE_TEST_TIER_3=1` | ~‚Ç¨0.01‚Äì0.05/run | instance(+lifecycle), instance-pool(+scale), compute-snapshot, block-volume attach/detach |
| 4 ‚Äî expensive / long-lived | `EXOSCALE_TEST_TIER_4_DBAAS=1`, `EXOSCALE_TEST_TIER_4_SKS=1`, `EXOSCALE_TEST_TIER_4_LB=1` | each ~‚Ç¨0.01‚Äì0.10/run, 10‚Äì20 min | dbaas, sks(+nodepool+kubeconfig), load-balancer(+service) |
| ‚ùå ‚Äî out of live scope | n/a | n/a | iam-user create (sends invite emails) |

All tiers require `EXOSCALE_ALLOW_MUTATION=1` as the master switch.

## Per-asset plan

For every asset type below: **Ops** = the methods the connector currently
exposes, **Live sequence** = the exact provisioning/verify/mutate/cleanup
script, **Cost/risk** = what we burn and what could go wrong, **Tier** = the
flag gate. Decide one row at a time ‚Äî anything you say no to, I skip.

### Tier 1 ‚Äî free mutations

#### 1. `security-group` (+ rules)
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`, `add_rule`, `delete_rule`.
- **Live sequence**:
  1. `create({"name": <prefix>-sg, "description": "connector smoke"})`
  2. `get` by id; `find_by_name`; assert round-trip.
  3. `add_rule` ingress tcp/443 from `0.0.0.0/0` with description `<prefix>-rule`.
  4. `get` again; assert rule in `.rules`.
  5. `delete_rule(sg_id, rule_id)`; assert rule gone.
  6. `delete(sg_id)`.
- **Cost**: ‚Ç¨0. **Risk**: none ‚Äî SGs are free metadata.

#### 2. `private-network`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete` *(no attach/detach helper yet in connector)*.
- **Live sequence**: create unmanaged L2 PN `<prefix>-pn` ‚Üí get ‚Üí update description ‚Üí delete.
- **Cost**: ‚Ç¨0. **Risk**: none.

#### 3. `anti-affinity-group`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `delete` *(no update ‚Äî API doesn't allow)*.
- **Live sequence**: create `<prefix>-aag` ‚Üí get ‚Üí list ‚Üí delete.
- **Cost**: ‚Ç¨0. **Risk**: none.

#### 4. `ssh-key`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `delete` *(no update ‚Äî keys are immutable)*. id_field = name.
- **Live sequence**: generate an ephemeral ed25519 keypair in-memory (test-only, never written to disk) ‚Üí register public key as `<prefix>-key` ‚Üí get by name ‚Üí delete by name.
- **Cost**: ‚Ç¨0. **Risk**: none. The private key never leaves the test process.

#### 5. `iam-role`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`.
- **Live sequence**: create role `<prefix>-role` with a minimal policy (deny-all default + permit `compute/listInstances` only) ‚Üí get ‚Üí update description ‚Üí delete.
- **Cost**: ‚Ç¨0. **Risk**: roles are scoped to the account. The role we create has zero effective permissions outside `listInstances`; even if leaked, harmless.

#### 6. `dns` (domain + records)
- **Ops**: domain CRUD (`list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`); record CRUD (`list_records`, `get_record`, `create_record`, `update_record`, `delete_record`).
- **Live sequence**:
  1. Create zone `<prefix>.test` (a name we don't own ‚Äî won't resolve publicly, that's fine).
  2. Create records: A `www` ‚Üí `192.0.2.1`, TXT `_test` ‚Üí `"hello"`, MX `@` ‚Üí `10 mail.<prefix>.test`.
  3. List/get records.
  4. Update the A record TTL.
  5. Delete each record.
  6. Delete the zone.
- **Cost**: Exoscale DNS may charge per zone (verify your account; typically ~‚Ç¨0/month for unused zones, or single-digit cents). **Risk**: zone is isolated.

### Tier 2 ‚Äî cheap, no compute

#### 7. `elastic-ip`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`. (No reverse-DNS-reset helper in connector yet.)
- **Live sequence**: create EIP with description `<prefix>-eip` ‚Üí get ‚Üí update description ‚Üí delete.
- **Cost**: EIP is charged ~‚Ç¨0.0027/hr while allocated; ~‚Ç¨0 for a few minutes.
- **Risk**: none. The address is released on delete.

#### 8. `bucket` (Object Storage ‚Äî S3 via boto3)
- **Ops**: `list`, `exists`, `create`, `delete`.
- **Prereq**: `boto3` installed (`pip install '.[sos]'`).
- **Live sequence**:
  1. Bucket name `<prefix>-bucket-<16-char-random>` (globally unique).
  2. `create` ‚Üí `exists` ‚Üí `list` (assert presence) ‚Üí `delete`.
- **Cost**: ~‚Ç¨0 (empty bucket for minutes).
- **Risk**: bucket name collision ‚Äî mitigated by 16-char random suffix. `delete_bucket` fails if non-empty; we never write objects, so safe.

#### 9. `block-volume` (detached only ‚Äî attach + resize live in Tier 3)
- **Ops** (this tier): `list`, `get`, `find_by_name`, `get_or_none`, `create`, `delete`, `create_snapshot`.
- **Live sequence**:
  1. Create 10 GB volume `<prefix>-vol` (smallest size) in test zone.
  2. Get; `find_by_name`; assert size + `state == "detached"`.
  3. `create_snapshot` ‚Üí tracks a `block-volume-snapshot` id (deleted in step 4).
  4. Delete snapshot, then delete volume.
- **Note**: `resize`, `attach` and `detach` are online operations ‚Äî the volume
  has to be attached to a running instance for the API to actually propagate
  the change. They move to Tier 3 where an instance fixture is available.
- **Cost**: ~‚Ç¨0.001/hr for 10 GB; cents per run.
- **Risk**: must delete snapshot before volume.

#### 10. `block-volume-snapshot`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `delete`, `update`, `create_from_volume`.
- **Live sequence**: covered inside the block-volume test above (step 4‚Äì6).

### Tier 3 ‚Äî compute, runs minutes (single-digit cents per run)

For everything here, the instance type is hardcoded to **`standard.tiny`** (the
smallest available, ~‚Ç¨0.0054/hr) and the disk to **10 GB**. The template is the
smallest current Linux LTS the account has access to (resolved at fixture time,
not hardcoded).

#### 11. `instance` (+ lifecycle)
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`, `start`, `stop`, `reboot`.
- **Live sequence**:
  1. Prereqs: ssh-key (`<prefix>-key`) + security-group (`<prefix>-sg`) ‚Äî created and torn down by Tier 1 fixtures.
  2. `create` `<prefix>-instance` (type `standard.tiny`, disk 10 GB, no public-IP if API allows, attached to the SG + ssh-key above).
  3. Wait until `state == "running"`; `get`; assert fields.
  4. `update` labels (`{"role": "test"}`).
  5. `stop` ‚Üí wait `stopped` ‚Üí `start` ‚Üí wait `running` ‚Üí `reboot` ‚Üí wait `running`.
  6. **Upscale**: stop ‚Üí change type to `standard.medium` via `update` (or the resize-action endpoint, whichever the API expects ‚Äî verified at run) ‚Üí start ‚Üí re-get ‚Üí assert type changed ‚Üí stop ‚Üí change back to `standard.tiny`.
  7. `delete`; wait gone.
- **Cost**: ~10‚Äì15 min √ó ‚Ç¨0.0054/hr ‚âà ‚Ç¨0.001‚Äì0.002. Plus ~‚Ç¨0.001 for the medium step.
- **Risk**: longest single-test runtime (~10 min); slowest cleanup. The `EXOSCALE_TEST_TIER_3=1` flag controls all of this.

#### 12. `instance-pool` (+ scale)
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`, `scale`.
- **Live sequence**:
  1. `create` `<prefix>-pool` with `size=1`, type `standard.tiny`, 10 GB.
  2. Wait until 1 member is running.
  3. `scale` to 2; wait; assert size==2.
  4. `update` description.
  5. `scale` back to 1.
  6. `delete` pool; verify members are terminated.
- **Cost**: peaks at 2 √ó tiny for ~2 min ‚Üí ~‚Ç¨0.001.
- **Risk**: pool deletion cascades to member instances ‚Äî wait for cascade before reusing names.

#### 13. `snapshot` (compute snapshot)
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `delete`, `create_from_instance`, `export`.
- **Live sequence** (runs while the Tier 3 instance from #11 exists):
  1. `create_from_instance(instance_id)`; wait until `state` is terminal ‚Äî
     current Exoscale auto-exports snapshots so the terminal state is
     `"exported"` (an earlier version assumed `"ready"`, which the Tier 3 run
     corrected).
  2. `get`; `list`; assert.
  3. `delete` snapshot.
- **Not yet live-verified:** `export` (the action endpoint that returns a
  pre-signed download URL) ‚Äî exposed on the client, mocked-tested, but the
  Tier 3 live test does not yet call it. Add to coverage in a follow-up if
  needed.
- **Cost**: snapshot storage per GB-month; 10 GB for minutes ‚âà ‚Ç¨0.
- **Risk**: snapshot creation takes 2‚Äì5 min on a 10 GB disk.

#### 13a. `block-volume` online operations (attach / resize / detach)
- **Ops** (Tier 3 additions): `attach`, `resize`, `detach`.
- **Live sequence** (runs while the Tier 3 instance from #11 exists):
  1. Create a 10 GB volume `<prefix>-vol-online`.
  2. `attach(volume_id, instance_id)`; wait until `state == "attached"`.
  3. `resize(volume_id, 20)`; re-get; assert `size == 20`.
  4. `detach(volume_id)`; wait until `state == "detached"`.
  5. Delete the volume.
- **Cost**: same as Tier 2 volume (~‚Ç¨0.001/hr) plus negligible attach overhead.
- **Risk**: must detach before delete; resize only grows.

### Tier 4 ‚Äî expensive / long-lived (each gated separately)

#### 14. `load-balancer` (+ services) ‚Äî `EXOSCALE_TEST_TIER_4_LB=1`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`, `add_service`, `update_service`, `delete_service`.
- **Prereq**: an `instance-pool` to point a service at (creates one inline if Tier 3 isn't already running).
- **Live sequence**:
  1. Create LB `<prefix>-lb` (smallest NLB profile).
  2. Create `<prefix>-pool-lb` (size=1, standard.tiny) as the service backend.
  3. `add_service` port 80 ‚Üí backend port 80, healthcheck tcp/80.
  4. Get LB; assert service appears under `.services`.
  5. `update_service` (change healthcheck interval).
  6. `delete_service`.
  7. `delete` LB.
  8. Delete the backing pool.
- **Cost**: NLB basic ~‚Ç¨0.0185/hr; pool tiny ~‚Ç¨0.0054/hr; ~10 min run ‚âà ‚Ç¨0.005.
- **Risk**: LB and pool must both be cleaned. The fixture tracks both ids.

#### 15. `dbaas` ‚Äî `EXOSCALE_TEST_TIER_4_DBAAS=1`
- **Ops**: `list`, `get`, `find_by_name`, `get_or_none`, `create` (type-specific), `update`, `delete`, `get_connection_info`, `reveal_user_password`, `list_service_types`.
- **Live sequence**:
  1. `list_service_types()`; pick the cheapest **pg** plan dynamically (typically `hobbyist-2` or equivalent ‚Äî resolved at run time, not hardcoded).
  2. `create(service_type="pg", name="<prefix>-pg", payload={"plan": <cheapest>, "version": <stable>})`.
  3. Wait until `state == "running"` (5‚Äì10 min).
  4. `get`; assert; `get_connection_info` ‚Äî assert host/port present, **never print the credentials**.
  5. `reveal_user_password("avnadmin")` ‚Äî assert non-empty, **never print**.
  6. `update` maintenance window or `ip-filter` to a no-op change.
  7. `delete` (`POWEROFF`-then-`DELETE` if required by API; verified at run).
- **Cost**: cheapest pg plan is hourly; ~15 min ‚âà ‚Ç¨0.01‚Äì0.05 depending on plan tier in your account.
- **Risk**: longest provisioning of any tier; secrets in responses (we assert presence, never log/print).

#### 16. `sks` (cluster + nodepool + kubeconfig) ‚Äî `EXOSCALE_TEST_TIER_4_SKS=1`
- **Ops**: cluster CRUD (`list`, `get`, `find_by_name`, `get_or_none`, `create`, `update`, `delete`); nodepool CRUD (`list_nodepools`, `get_nodepool`, `create_nodepool`, `update_nodepool`, `delete_nodepool`); `generate_kubeconfig`.
- **Live sequence**:
  1. `create` cluster `<prefix>-sks` (service-level `starter`, smallest version available, CNI `calico`).
  2. Wait `state == "running"` (5‚Äì10 min).
  3. `create_nodepool(<cluster_id>, size=1, instance_type=standard.tiny, disk_size=20)`.
  4. Wait nodepool ready.
  5. **Upscale**: `update_nodepool` size to 2; wait; assert; `update_nodepool` size back to 1.
  6. `generate_kubeconfig` for a temp user `<prefix>-kube`; assert kubeconfig YAML present, **don't print**.
  7. `delete_nodepool`.
  8. `delete` cluster.
- **Cost**: starter control plane is free; 1 √ó standard.tiny for ~15 min ‚âà ‚Ç¨0.001; brief 2x during upscale.
- **Risk**: longest total runtime (~15 min). Cluster delete must complete before nodepool ids release.

### Out of live scope

#### `iam-user` create / update / delete ‚Äî **disabled in live tests**
- **Reason**: creating an org user triggers an invite email to a real address. Even with a placeholder, this either fails (bounce) or spams someone. The mocked tests cover request shaping; live mutation is not safe.
- **Live coverage**: read-only only ‚Äî `list`, `get`, `find_by_name` on existing users to confirm parsing.
- Reconsider only if you supply a controlled mailbox you don't mind receiving 1 invite per test run.

#### `api-key` create ‚Äî **gated, off by default**
- The API key create works without email side-effects, but the **secret is returned exactly once** and would briefly exist as an attachment to a role.
- Plan: create key bound to the Tier 1 `<prefix>-role`, assert `.secret` is non-empty (without printing it), then immediately `delete` both the key and the role.
- Gated separately: `EXOSCALE_TEST_TIER_1_API_KEY=1` (off even within Tier 1 unless you flip this).

## Cost ceiling ‚Äî worst case (everything enabled)

If every tier is enabled and a single full run completes successfully:

| Tier | Worst-case cost per full run |
|------|----------------------------|
| 0 | ‚Ç¨0 |
| 1 | ‚Ç¨0 |
| 2 | < ‚Ç¨0.01 |
| 3 | < ‚Ç¨0.05 (most of this is the instance type-change cycle) |
| 4 LB | < ‚Ç¨0.01 |
| 4 DBaaS | ‚Ç¨0.01‚Äì0.05 (depends on cheapest plan in your account) |
| 4 SKS | < ‚Ç¨0.01 |
| **Total worst case** | **~‚Ç¨0.10 per full run** |

If a run *crashes* mid-way and orphans the most expensive thing (a DBaaS
service), the daily cost is whatever that plan charges per day until the orphan
collector or you delete it. The session-end sweep + the optional GC script are
designed to make this rare.

## Test execution shape

Every per-asset test follows this skeleton:

```python
def test_<asset>_lifecycle(live_client, run_id, tracker):
    # 1. PROVISION
    created = <Client>(live_client).create(payload=<minimal_payload(run_id)>)
    tracker.register(<Client>, created.id)        # for guaranteed teardown

    # 2. VERIFY
    fetched = <Client>(live_client).get(created.id)
    assert_minimum_fields(fetched)

    # 3. MUTATE (where the API supports it)
    <Client>(live_client).update(created.id, {...})
    <Client>(live_client).<extra_op>(...)         # start/stop/scale/resize/...

    # 4. CLEANUP
    <Client>(live_client).delete(created.id)
    tracker.unregister(<Client>, created.id)
```

The session finalizer iterates the tracker; anything left is a leak and is
deleted with the safety-prefix guard.

## Pre-flight checklist (before any live run)

- [ ] You confirm the credentials in the chosen secret manager point at a
      **test/staging tenant**, not production.
- [ ] `EXOSCALE_API_KEY` / `EXOSCALE_API_SECRET` are injected into the
      environment; no secrets on the command line.
- [ ] `EXOSCALE_ZONE` set to a zone you actually want to provision in.
- [ ] `EXOSCALE_RUN_LIVE_TESTS=1`.
- [ ] Tier flags set explicitly for what you approved.
- [ ] `EXOSCALE_ALLOW_MUTATION=1` *only* for the tiers you approved.
- [ ] No leftover `conn-test-*` resources from a prior crashed run. Quick
      manual check: `exoscale-<asset> list | jq '.[] | select(.name | startswith("conn-test-"))'`
      for the asset types you're about to test, or run the orphan-cleanup
      snippet at the top of [live-test-results.md](live-test-results.md).
